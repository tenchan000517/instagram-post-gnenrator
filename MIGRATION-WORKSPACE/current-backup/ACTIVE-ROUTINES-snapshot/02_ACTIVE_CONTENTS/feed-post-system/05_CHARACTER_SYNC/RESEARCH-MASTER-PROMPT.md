# フィードポスト投稿用統合データベースリサーチシステム

## 🎯 システム概要
既存の3つのデータベースシステムをフィードポスト生成に完全統合。テーマ入力から最適なワークフロー自動実行まで一貫したリサーチプロセスを提供。

**対応データベース**:
- 習慣行動データベース（habit-behavior-database.json + 分類DB 9種）
- 企業情報データベース（companyMasterData.json + rankingGenerator.js）
- 生産性ツールデータベース（productivity-tools-database.json）

---

## 🔄 自動振り分けシステム

### テーマ分類・最適フロー選択
```
【入力】投稿テーマ: [ユーザー入力テーマ]

【自動判定システム】
┌─ 習慣・行動系キーワード検出
│   → HABIT-DATABASE-RESEARCH.md実行
│   → habit-behavior-database.json活用
│
├─ 企業・就活系キーワード検出  
│   → COMPANY-DATABASE-RESEARCH.md実行
│   → companyMasterData.json + rankingGenerator.js活用
│
├─ 効率化・ツール系キーワード検出
│   → PRODUCTIVITY-DATABASE-RESEARCH.md実行
│   → productivity-tools-database.json活用
│
└─ その他・新規テーマ
    → NEW-RESEARCH-PROMPT.md実行
    → 新規リサーチ→データベース拡張
```

## 🚀 各データベース別実行フロー概要

### 1️⃣ 習慣データベース活用フロー
```
【対象】朝活、時間管理、学習習慣、健康習慣等
【活用DB】habit-behavior-database.json + 9分類DB
【既存ワークフロー】4ステップ起動術式を活用
【生成キャラ】主にType001(MISAKI), Type002(KING), Type004(TEN)
【アウトプット】習慣改善型フィードポスト
```

### 2️⃣ 企業データベース活用フロー  
```
【対象】年収ランキング、就活企業、業界分析等
【活用DB】companyMasterData.json + rankingGenerator.js
【既存ワークフロー】5Phase構築フロー + Type003生成プロセス
【生成キャラ】主にType003(KIKUYO)
【アウトプット】企業分析型フィードポスト
```

### 3️⃣ 生産性ツールデータベース活用フロー
```
【対象】効率化アプリ、ツール比較、自動化等
【活用DB】productivity-tools-database.json
【既存ワークフロー】TEN特化型分析プロセス
【生成キャラ】主にType004(TEN)
【アウトプット】効率化特化型フィードポスト
```

### 4️⃣ 新規テーマリサーチフロー
```
【対象】既存DB外の全テーマ
【プロセス】God of Research + 5専門家分析
【品質保証】Constitutional AI + Self-Consistency検証
【生成キャラ】テーマに応じてType001-005から最適選択
【アウトプット】新規テーマ対応フィードポスト
```
