# リール→フィード変換アンチョコ（簡易版）

## 🎯 基本原則
リールコンテンツ（10行程度）を、フィード投稿（8ページ構成）に拡張変換する

---

## 📋 必要ファイル
```
【元コンテンツ】
社会人向け：/03_ANALYSIS_DATA/x-post-analysis/contents/社会人向けコンテンツ100個02.md
女性向け：/03_ANALYSIS_DATA/x-post-analysis/archive/女性向けコンテンツ100個02.md  
学生向け：/03_ANALYSIS_DATA/x-post-analysis/contents/学生向けコンテンツ100個02.md
```

---

## 🔄 変換マッピング

| 元コンテンツ | 変換先Type | ターゲット | キャラクター |
|------------|-----------|----------|------------|
| 社会人向け（KING） | Type002 | T007, T010 | 男性的・力強い |
| 女性向け（MISAKI） | Type001 | T001-T003 | 女性的・共感的 |
| 学生向け（IIDA） | Type002 | T007-T009 | 誠実・サポート |

---

## 📝 変換手順（5ステップ）

### 1️⃣ コア価値抽出
リールコンテンツから「核心メッセージ」を特定
```
例：「80%の力で仕事」→「完璧主義をやめる価値」
```

### 2️⃣ Type別構造設計
- **Type001**：感情共感型（section_blocks構造）
  - 共感→問題分析→希望→解決策
- **Type002**：実践習得型（step_by_step構造）  
  - 理解→実践→習慣化→成果

### 3️⃣ 8ページ展開
```json
Page1: 問題提起（タイトル・悩み）
Page2-7: Type別展開
  Type001: 共感2P→分析1P→希望1P→解決2P
  Type002: 理解1P→実践4P→習慣化1P  
Page8: まとめ（成果・CTA）
```

### 4️⃣ 深掘り要素追加
- **統計データ**：説得力向上
- **具体例**：3-5個の実例
- **実践手順**：ステップバイステップ
- **視覚要素**：HTMLビジュアル

### 5️⃣ 品質チェック
- 禁止表現排除（「効率的」「最適化」等）
- 自然な日本語（AI臭さ回避）
- ターゲット性別一致確認
- 99%品質基準クリア

---

## 💡 変換のコツ

### ✅ DO
- リールの「感情」を大切に拡張
- 具体的な数字・事例を追加
- 読者の「明日やってみよう」を意識
- キャラクターらしさを保持

### ❌ DON'T  
- 単純な文章引き伸ばし
- 機械的な情報追加
- 元の温度感を失う
- Type違反の構造使用

