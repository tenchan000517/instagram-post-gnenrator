### ✅ 確立したシステム

**基本理解**：
- 1つのコンテンツ（完成された1ページ）から4つの独立した投稿を作成
- 各投稿は+-×÷タイプで5～10選のリスト形式
- コンテンツファイルは複数存在（01.md、02.md、03.md...）

**作業フロー**：
1. コンテンツ読み込み
2. 成功定義（視聴者にどうなってほしいか）
3. 事例参照（Sランク・Aランク事例集）
4. タイトル作成（+-×÷の4つ）
5. 自己評価（頻出チェック・事例比較チェック）
6. 修正
7. 出力（複数コンテンツ同時出力形式）

## 🔧 システムの設計思想

### 基本原則

1. **事例優先主義**
   - 理論や法則より実際の成功事例を重視
   - Sランク事例（実際のInstagram投稿）が最高基準
   - Aランク事例（検証済みOKタイトル）で補完

2. **ワードセンス重視**
   - 日常的で身近な言葉を使用
   - 「やってる」「した」「良かった」など自然な動詞
   - 抽象的表現の排除

3. **品質管理システム**
   - 頻出チェック（同じ表現の多用防止）
   - 事例比較チェック（突飛な表現の防止）

**効率性**：
- 5～10コンテンツ同時出力でファイル管理効率化

- 同じフロー・同じ品質管理で横展開可能

**実用性**：

## ⚠️ 重要な注意事項・継承すべき学び

### 1. 根本的理解（絶対に間違えてはいけない）

**コンテンツと投稿の関係**：
- コンテンツ = 完成された1ページ（触らない）
- 投稿 = コンテンツから作る4つの独立した投稿
- 1つのコンテンツ→4つの別々の投稿作成

**ファイル構造**：
- 複数のコンテンツファイル存在（01.md、02.md...）
- 各ファイル内に複数コンテンツ（コンテンツ1、2、3...）
- 出力時はファイル名を必ず明記

### 2. 品質管理の徹底

**頻出チェック（マスト）**：
- 同じ表現の乱用は絶対に避ける
- 特に-タイプのバリエーション重視
- 「思考パターン」「特徴」等の多用注意

**事例比較チェック**：
- Sランク事例と比較して突飛でないか
- 日常的な表現になっているか
- ワードセンスが自然か

### 3. 禁止事項

**やってはいけないこと**：
- センスのない状態でのアレンジ
- 理論優先での複雑化
- 抽象的表現の使用
- 事例を無視した創作
- 体系の無視

**センスについて**：
- 自分でセンスを身につけようとしない
- 成功事例のパターン・構造・ワードセンスに沿う
- アレンジは最小限に抑制
