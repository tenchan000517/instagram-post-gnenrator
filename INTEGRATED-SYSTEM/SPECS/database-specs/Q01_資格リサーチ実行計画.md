## 🔧 **各フェーズの詳細指示**

### **Phase 1: バッチリサーチ実行**

#### **エンジニアへの具体的指示**
1. **担当バッチの確定**
   ```bash
   # 例：Batch 3担当の場合
   cd app/data/qualificationDatabase/research-batches/
   touch batch-03-medical-welfare.json
   ```

2. **リサーチ対象の特定**
   - 医療・福祉カテゴリの20資格をリストアップ
   - 各資格の公式サイトを特定
   - リサーチ順序を決定

3. **データ収集実行**
   - 17項目を一つずつリサーチ
   - DATABASE_CONSTRUCTION_GUIDELINES.mdに従って入力
   - 5軸評価を算出

4. **品質チェック**
   ```javascript
   // validateData.jsを実行
   node scripts/validateData.js batch-03-medical-welfare.json
   ```

### **Phase 2: データ統合**

#### **統合実行者への指示**
```bash
# 全バッチ完了後に実行
cd app/data/qualificationDatabase/
node scripts/integrateBatches.js

# 実行結果
# → qualificationMasterData.json生成
# → 統合レポート出力
# → 重複チェック結果表示
```

#### **統合スクリプトの作成要件**
- 各バッチファイルの読み込み
- IDの重複チェック
- カテゴリ名の統一
- データフォーマットの検証
- 統合データベースの生成

### **Phase 3: スコア計算・検証**

#### **実行指示**
```bash
# スコア再計算（念のため）
node scripts/calculateScores.js

# データ品質最終チェック
node scripts/validateData.js qualificationMasterData.json
```

### **Phase 4: ランキング生成**

#### **ランキング生成の指示**
```bash
node scripts/generateRankings.js

# 生成されるランキング例
# - 講習のみで取得可能TOP15
# - 取得コスト最安TOP20
# - 収入インパクト最高TOP10
# - 専門性最高TOP10
```

### **Phase 5: ナレッジファイル作成**

#### **KIKUYO用ナレッジファイル作成**

### **Phase 6: リール台本作成**

## ⚠️ **重要な注意事項**

### **データ品質保証**
1. **各バッチ担当者は必ずvalidateData.jsを実行**
2. **不明な項目は「要確認」として明記**
3. **推測データは使用禁止**
4. **公式情報のみを使用**

### **進捗共有**
- 各バッチ完了時に進捗管理表を更新
- 問題発生時は即座に報告
- 他バッチとの整合性を随時確認

### **品質基準**
- データ精度95%以上
- 必須項目の入力率100%
- スクリプト実行エラー0件

## 🚨 **緊急時対応**

### **バッチ作業中の問題**
1. **データソース不明**: 一時保留して他の資格を進める
2. **評価判断困難**: ガイドラインの具体例を参照
3. **技術的問題**: 他のエンジニアまたは管理者に相談

### **統合時の問題**
- ID重複: 自動的に連番で解決
- フォーマット不一致: エラー表示して該当バッチを特定
- データ欠損: 欠損項目一覧を出力
