# 04 追加資格リサーチ実行（運用フェーズ用）

**対象**: 完成したデータベースへの新資格追加  
**手法**: 単体リサーチ→統合→再生成  
**前提**: 121資格データベースが完成済み  
**保存先**: `app/data/qualificationDatabase/`

---

## 🎯 **使用タイミング**

### **このドキュメントを使う場面**
- 資格データベース（121資格）が完成している
- 新しい資格（例：静岡県検定）を追加したい
- ランキングやナレッジファイルも更新したい

### **使わない場面**
- 初回の121資格リサーチ（`03_資格リサーチ実行_V2025.md`を使用）
- 大量追加（10個以上なら初回バッチ方式が効率的）

---

## 🔧 **実行手順**

### **STEP 1: 新資格ID確定**
```bash
# 現在の最後のIDを確認
grep -o '"id": "Q[0-9]*"' app/data/qualificationDatabase/qualificationMasterData.json | tail -1
# 例：Q121なら、新資格はQ122
```

### **STEP 2: 必須ドキュメント確認**
追加する資格について、以下を参照してリサーチを実行：
1. **ガイドライン**: `INTEGRATED-SYSTEM/SPECS/database-specs/Q02_資格データベース構築ガイドライン.md`
   - 17項目の詳細定義
   - 5軸評価の定量基準
   - データ入力の絶対ルール

### **STEP 3: 単体リサーチ実行**
```bash
# 作業ディレクトリ準備
cd app/data/qualificationDatabase/
mkdir -p additions/

# 新資格ファイル作成
# 例：静岡県検定の場合
touch additions/Q122_静岡県検定.json
```

### **STEP 4: データ作成**
ガイドラインに従って以下を作成：
- **17項目の詳細調査**
- **5軸評価スコア算出**
- **指定JSONフォーマット**

### **STEP 5: 既存データベースに統合**
```bash
# 統合スクリプト実行
node scripts/addNewQualification.js additions/Q122_静岡県検定.json

# 結果確認
# → qualificationMasterData.jsonが更新される
# → 総資格数が121→122になる
```

### **STEP 6: ランキング再生成**
```bash
# 全ランキングを再計算
node scripts/generateRankings.js

# 影響するランキング例：
# - 講習のみで取得可能TOP15
# - 取得コスト最安TOP20
# - 収入インパクト最高TOP10
```

### **STEP 7: ナレッジファイル更新**
追加した資格が以下に該当する場合、ナレッジファイルを更新：
- **講習系資格**: KYxxxファイル（KIKUYO用）への影響確認
- **ランキング変動**: 該当するナレッジファイルの更新

## ⚠️ **注意事項**

### **大量追加の場合**
5個以上の資格を一度に追加する場合は、初回バッチ方式（`03_資格リサーチ実行_V2025.md`）を参考に効率化を検討

### **データ品質**
- **単体追加でも品質基準は同じ**
- 17項目の完全入力必須
- 5軸評価の定量基準遵守

### **影響範囲**
- ランキング順位の変動可能性
- 既存ナレッジファイルの更新要否
- リール動画用素材への影響
