## 🛠️ 統合作成手順（6ステップ）

### **STEP1: 基盤情報収集・最適化準備**
- [ ] 指定されたコンテンツ番号の元コンテンツを読み取り
- [ ] 対応する4タイプタイトルを確認
- [ ] 元コンテンツの核心メッセージを理解
- [ ] **最適化モード起動**: 最終品質基準を適用

### **STEP2: 9ページ構成設計（最適化済み）**
**基本構成**:
```
1ページ目: メインタイトル
2ページ目: 元コンテンツ（MISAKIらしい優しい言葉・簡潔化）
3ページ目: 比較表（リスト形式・具体化）
4ページ目: 改めてタイトル
5-9ページ目: コツページ（最適化済みタイトル・一般化済み内容）
```

### **STEP3: 最適化比較表作成**
**形式**:
```
[目標などの対象]が達成できない人
[目標などの対象]達成した人

[−タイプタイトル]
項目1（動詞を含む具体的行動）
項目2（動詞を含む具体的行動）
...項目8

[÷タイプタイトル] 
項目1（動詞を含む具体的結果）
項目2（動詞を含む具体的結果）
...項目8
```

**最適化ルール**:
- [ ] 各項目に動詞を追加（「やりたくない事」→「やりたくない事は避ける」）
- [ ] 対比が明確になる表現
- [ ] 12文字以内で調整
- [ ] 行動の具体化

### **STEP4: 最適化コツページ作成（5-9ページ）**

#### **タイトル最適化4原則（自動適用）**
```
1. 修飾語削除：「難しい」「チャレンジングな」等除去
2. 動詞直接化：「手を挙げる」→「挑戦する」
3. 主語省略：「上司に」→省略（文脈で理解可能）
4. 10-12文字以内：読みやすさ重視
```

#### **最適化ページ構成テンプレート**
```
### Xページ目: +タイトル+習慣X
[超簡潔タイトル]（10-12文字・最適化済み）

[説明文4行・一般化済み]
- 1行目: 状況設定（「〜と思いますが」）
- 2行目: 行動提案（「〜してみると」「〜すると」）
- 3行目: 結果・効果（「〜できて」「〜し」）
- 4行目: 追加効果（「〜します」「〜されます」）
```

#### **自動適用ルール**
- [ ] **体験談完全一般化**: 「〜した」→「〜します」「〜できます」
- [ ] **表現自然化**: 「ですね」→「です」、「上司から」→「周囲から」
- [ ] **能動的表現**: 意図的・自分から行動する内容
- [ ] **MISAKIトーン**: 温かい・優しい・共感的

### **STEP5: 最適化ナレーション作成**

#### **構造（簡潔最適化版）**
```
段落1: 導入（40-60文字）
[元コンテンツ核心]って思う時、[重要ポイント]ですよね。[効果的アプローチ]なんです。

段落2: 効果説明（20-30文字）  
[良い習慣の効果]。

段落3: 対比説明（60-80文字）
[できない人特徴]になりがち。一方で[できる人特徴]。

段落4: タイトル提示（1回のみ）
[+タイトル]。

段落5: 5つの具体例（各30-40文字・超簡潔）
まず[習慣1簡潔]。[不安表現]けど、[結果]。
次に[習慣2簡潔]。[状況]時も、[効果]んです。
３つめは[習慣3簡潔]。[行動理由]ことで[結果]。
４つめは[習慣4簡潔]。[期間]けど[結果]に。
さいごは[習慣5簡潔]。[印象]けど[継続効果]。

段落6: CTA（30-40文字）
参考になったと思ってもらえたらいいねやコメントで教ってもらえるとうれしいです！
```

#### **ナレーション最適化ルール**
- [ ] **タイトル重複排除**: 2回提示→1回のみ
- [ ] **超簡潔化**: 各習慣1-2文
- [ ] **接続詞削減**: 「そして」「また」等の除去
- [ ] **総文字数530文字以内**
- [ ] **テンポ重視**: 読み上げやすさ確保

### **STEP6: キャプション生成**
**重要**: 必ず以下のマニュアルを熟読してから作成
```
INTEGRATED-SYSTEM/RESOURCES/creation-manuals/10_キャプション・ハッシュタグ生成マニュアル_2025-08-23.md
```
- [ ] マニュアルの「MISAKIタイプ」セクションを確認
- [ ] 正しいフォーマットで生成（顔文字は最小限）

### **STEP7: 統合完成・保存**
- [ ] ナレーション最適化完成
- [ ] キャプション生成完了（マニュアル準拠）
- [ ] 最終品質チェック完了
- [ ] ファイル保存: `INTEGRATED-SYSTEM/OUTPUT/reel-posts/MISAKI/misaki_content_XXX.md`

## 🔧 トラブルシューティング

### **品質レベル不足の場合**
**症状**: 初稿レベルで作成されてしまう
**対処**: STEP1で「最適化モード起動」を必ず確認

### **タイトルが長すぎる場合**
**症状**: 15文字以上になる
**対処**: 4原則を再適用（修飾語削除・動詞直接化・主語省略）

### **ナレーションが冗長な場合**
**症状**: 600文字を超える
**対処**: 段落5で各習慣を1-2文に強制簡潔化
