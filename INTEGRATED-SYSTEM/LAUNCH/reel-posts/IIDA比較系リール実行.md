## 🛠️ IIDA比較系作成手順（7ステップ）

### **STEP1: 比較要素抽出・最適化準備**
- [ ] 統合最終版リストから指定タイトルを確認
- [ ] 対応するロードマップ章から比較可能な実例を抽出
- [ ] 良い例・悪い例の具体的事例を特定
- [ ] **比較モード起動**: いい例 vs 悪い例の明確化

### **STEP2: 9ページ構成設計（IIDA比較版）**
**基本構成**:
```
1ページ目: 質問
2ページ目: 悪い例
3ページ目: 質問  
4ページ目: いい例
5ページ目: 質問
6ページ目: 悪い例
7ページ目: 質問
8ページ目: いい例
9ページ目: 質問
```

### **STEP3: IIDA版実例比較構成**
**ロードマップ実例活用ルール**:
- [ ] 内定者の実際の回答全文を「いい例」として使用
- [ ] よくある失敗例を「悪い例」として具体化
- [ ] 企業名・面接官の反応など具体的情報含有
- [ ] そのまま参考にできるテンプレート提供

**比較フォーマット**:
```
【3ページ目：❌ 悪い例】
よくあるNG例
「[実際のNG回答全文]」
→ [面接官の反応・問題点]

【4ページ目：⭕ いい例】  
[企業名]内定者の実例
「[実際の成功回答全文]」
→ [面接官の反応・高評価理由]
```

### **STEP4: IIDA比較ページ作成（5-8ページ）**

#### **比較ポイント抽出原則（IIDA版）**
```
1. 具体性：「表現が曖昧」→「数字・事実が含まれている」
2. 構造：「だらだら話す」→「結論→理由→具体例の順」
3. 印象：「自信がなさそう」→「堂々と話している」
4. 内容：「ありきたり」→「その人らしさが伝わる」
```

#### **IIDA比較ページ構成テンプレート**
```
### Xページ目: 違いのポイントX
[具体的違いタイトル]（10-15文字・実例重視）

[説明文4行・比較重視]
- 1行目: 悪い例の問題（「NG例では〜になってしまう」）
- 2行目: いい例の良さ（「内定者は〜している」）
- 3行目: その効果（「だから〜という印象を与える」）
- 4行目: 実践方法（「あなたも〜を意識してみて」）
```

#### **IIDA比較自動適用ルール**
- [ ] **実例対比**: 抽象論→具体的成功・失敗例
- [ ] **共感表現**: 「失敗する人は」→「私も最初は失敗した」
- [ ] **励まし**: 「ダメ」→「改善できるから大丈夫」
- [ ] **IIDAトーン**: 親身・温かい・具体的指導

### **STEP5: IIDA比較ナレーション作成**

#### **構造（IIDA比較シンプル版）**
```
質問1の内容をそのまま読み上げ
→ 悪い例1の回答をそのまま読み上げ
→ 質問2の内容をそのまま読み上げ  
→ いい例1の回答をそのまま読み上げ
→ 質問3の内容をそのまま読み上げ
→ 悪い例2の回答をそのまま読み上げ
→ 質問4の内容をそのまま読み上げ
→ いい例2の回答をそのまま読み上げ
→ 質問5の内容をそのまま読み上げ
→ 悪い例3の回答をそのまま読み上げ
→ 質問6の内容をそのまま読み上げ
→ いい例3の回答をそのまま読み上げ
```

#### **IIDA比較ナレーション最適化ルール**
- [ ] **そのまま読み上げ**: 質問・回答例に解説や評価を追加しない
- [ ] **シンプル構成**: 質問→悪い例→質問→いい例の繰り返しのみ
- [ ] **総文字数530文字以内**
- [ ] **読みやすさ重視**: 淡々と読み上げやすい構成

### **STEP6: キャプション生成**
**重要**: 必ず以下のマニュアルを熟読してから作成
```
INTEGRATED-SYSTEM/RESOURCES/creation-manuals/10_キャプション・ハッシュタグ生成マニュアル_2025-08-23.md
```
- [ ] マニュアルの「💼 Iida（就活・キャリア系）」セクションを確認
- [ ] IIDA掴み文パターン（比較系に適したもの）を選択
- [ ] 固定フォーマット完全準拠

### **STEP7: 統合完成・保存**
- [ ] リール動画9ページ構成完成
- [ ] 悪い例・いい例の具体性確認
- [ ] IIDA比較トーン一貫性確認
- [ ] ナレーション530文字以内確認
- [ ] キャプション固定フォーマット準拠確認
- [ ] 保存先: `INTEGRATED-SYSTEM/OUTPUT/reel-posts/IIDA/iida_comparison_XXX.md`
