## 🛠️ IIDA統合作成手順（7ステップ）

### **STEP1: 基盤情報収集・最適化準備**
- [ ] 統合最終版リストから指定タイトルを確認
- [ ] 対応するロードマップ章を読み込み
- [ ] **タイトルと元コンテンツの整合性確認**: 
  タイトルが約束する内容が元コンテンツに実際に存在するかチェック
- [ ] **ミスマッチ時の対応**: 
  不一致の場合は元コンテンツにないものを創作せず、
  元コンテンツの内容に合致する別タイトルを選択
- [ ] 実例・事例・テンプレートを抽出
- [ ] **最適化モード起動**: IIDA最終品質基準を適用
- [ ] **パターン判定**: 事例集の場合は比較系リール実行を使用

### **STEP2: 9ページ構成設計（IIDA版）**
**基本構成（MISAKI構造準拠）**:
```
1ページ目: メインタイトル（IIDAらしい励まし表現）
2ページ目: 元コンテンツ（IIDAらしい親身な言葉・簡潔化）
3ページ目: 比較表（リスト形式・就活具体例）
4ページ目: 改めてタイトル（行動促進）
5-9ページ目: 実践ポイント（5つの具体的要素・実例重視）
```

### **STEP3: IIDA版実例抽出・構成**
**ロードマップ活用ルール**:
- [ ] 具体的な企業名・数字・データを活用
- [ ] 内定者の実際の例・テンプレートを使用
- [ ] そのまま使えるフォーマット提供
- [ ] 方法論ではなく実例のみ

**構成フォーマット**:
```
【3ページ目：実例紹介】
実際に[企業名]内定者が使った[具体例]

【5-9ページ目：実践ポイント】
ポイント1：[具体的実例]
ポイント2：[具体的実例] 
ポイント3：[具体的実例]
ポイント4：[具体的実例]
ポイント5：[具体的実例]
```

### **STEP4: IIDAページ作成（5-9ページ）**

#### **タイトル最適化原則（IIDA版）**
```
1. 実例重視：「〜の方法」→「〜の実例」
2. 具体性重視：「効果的な」→「Google内定者の」
3. 励まし追加：「〜するコツ」→「〜して内定勝ち取るコツ」
4. 10-15文字以内：読みやすさ重視
```


#### **IIDAページ構成テンプレート**
```
### Xページ目: 実践ポイントX
[実例重視タイトル]（10-15文字・励まし含む）

[説明文4行・実例重視]
- 1行目: 実際の状況（「[企業名]面接で〜」）
- 2行目: 内定者の行動（「実際に〜した」）
- 3行目: その結果（「〜で内定獲得」）
- 4行目: 活用方法（「あなたも〜してみて」）
```

#### **IIDA自動適用ルール（MISAKI構造参照）**
- [ ] **実例中心**: 方法論→具体的成功例
- [ ] **励まし表現**: 「〜できる」→「一緒に頑張ろう」
- [ ] **親近感**: 「〜です」→「〜だよ」「〜してみて」
- [ ] **IIDAトーン**: 親身・温かい・応援・伴走感
- [ ] **比較表形式**: できない人vsできる人の具体的対比

### **STEP5: IIDA最適化ナレーション作成**

**重要**: 必ず以下のマニュアルを熟読してから作成
```
INTEGRATED-SYSTEM/RESOURCES/creation-manuals/MISAKI_ナレーション作成_マニュアル.md
```

#### **構造（6段落・530文字以内）**
```
段落1: 導入（40-60文字）

段落2: 効果説明（20-30文字）  
[良い習慣の効果]。

段落3: 対比説明（60-80文字）

段落4: タイトル提示（1回のみ）
[メインタイトル]。

段落5: 5つの具体例（各30-40文字・超簡潔）
まず[習慣1簡潔]。[不安表現]けど、[結果]。
次に[習慣2簡潔]。[状況]時も、[効果]んです。

4つめは[習慣4簡潔]。[期間]けど[結果]に。
最後は[習慣5簡潔]。[印象]けど[継続効果]。

段落6: CTA（30-40文字）
一緒に内定勝ち取ろう！質問があればコメントで教えてね。
```

#### **IIDAナレーション最適化ルール**
- [ ] **マニュアル準拠**: MISAKI_ナレーション作成_マニュアルの構造に従う
- [ ] **共感・励まし強調**: 「〜だよね」「一緒に頑張ろう」
- [ ] **実例数値化**: 具体的な数字・企業名を使用
- [ ] **総文字数530文字以内**: 厳守
- [ ] **親しみやすさ重視**: 読み上げやすい自然な日本語
