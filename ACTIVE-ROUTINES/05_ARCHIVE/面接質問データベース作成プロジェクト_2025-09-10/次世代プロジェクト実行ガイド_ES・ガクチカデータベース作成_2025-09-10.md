## 📋 ESデータベース作成プロジェクト

### Phase 1: 企画・設計 (1-2時間)

#### 1.1 データソース準備
```
必要ファイル:
├── ES質問100問_カテゴリ分け.md
├── ES模範解答事例①志望動機系.md
├── ES模範解答事例②自己PR系.md
└── ES模範解答事例③ガクチカ系.md
```

#### 1.2 カテゴリ設計
- **志望動機系** (30問)
- **自己PR・強み系** (25問) 
- **ガクチカ・経験系** (20問)
- **将来ビジョン系** (15問)
- **価値観・人物像系** (10問)

#### 1.3 ES特有評価ポイント
- **具体性**: 数値・事実ベース
- **独自性**: 他の学生との差別化
- **論理性**: 一貫した主張
- **企業適合性**: 志望企業との親和性

### Phase 2: スクリプト開発 (2-3時間)

#### 2.1 基盤スクリプト作成
```bash
# 面接DBスクリプトをテンプレートとして複製
cp add_questions_to_json.js add_es_questions.js
cp extract_and_add_answers.js extract_es_answers.js
cp validate_database.js validate_es_database.js
cp add_improved_bad_answers.js add_es_bad_answers.js
```

#### 2.2 ES専用カスタマイズポイント
```javascript
// ESの評価ポイント例
const evaluationPoints = {
  "具体性": "数値や事実を用いた説得力",
  "独自性": "他者との差別化要素",
  "論理性": "根拠と結論の一貫性",
  "企業理解": "志望企業への深い洞察"
};
```

### Phase 3: コンテンツ作成 (3-4時間)

#### 3.1 模範解答の特徴
- **文字数制限対応**: 200字/400字版
- **業界別カスタマイズ**: IT/金融/商社/メーカー
- **PREP法適用**: Point, Reason, Example, Point

#### 3.2 ES特有悪い例パターン
```javascript
const esBadAnswerTypes = {
  "抽象的すぎる": "「頑張りました」「学びました」連発",
  "企業研究不足": "どの会社でも使える汎用的内容",
  "自己中心的": "企業メリットへの言及なし",
  "文字数調整失敗": "無意味な文章で文字稼ぎ"
};
```

### Phase 4: 品質管理 (1時間)

#### 4.1 ESバリデーション項目
- 文字数チェック（200字/400字）
- 業界別適合性
- PREP構造チェック
- NGワード検出

## 📋 ガクチカデータベース作成プロジェクト

### Phase 1: 企画・設計 (1-2時間)

#### 1.1 データソース準備
```
必要ファイル:
├── ガクチカテーマ100選_分類.md
├── ガクチカ模範事例①アルバイト系.md  
├── ガクチカ模範事例②サークル・部活系.md
├── ガクチカ模範事例③学業・研究系.md
├── ガクチカ模範事例④ボランティア系.md
└── ガクチカ模範事例⑤その他活動系.md
```

#### 1.2 カテゴリ設計
- **アルバイト・インターン系** (30問)
- **サークル・部活系** (25問)
- **学業・研究・資格系** (20問)
- **ボランティア・社会活動系** (15問)
- **個人的挑戦・趣味系** (10問)

#### 1.3 ガクチカ特有評価ポイント
- **STAR法完備**: Situation, Task, Action, Result
- **再現性**: 社会人でも活かせるスキル
- **主体性**: 受け身でない能動的行動
- **成長性**: 失敗からの学びと改善

### Phase 2: スクリプト開発 (2-3時間)

#### 2.1 STAR法チェック機能
```javascript
const starValidation = {
  "Situation": "状況・背景の説明があるか",
  "Task": "自分の役割・目標が明確か", 
  "Action": "具体的な行動が記述されているか",
  "Result": "数値化された成果があるか"
};
```

#### 2.2 ガクチカ専用悪い例パターン
```javascript
const gakuchikaProblems = {
  "受け身すぎる": "指示されたことをこなしただけ",
  "スケール小さい": "個人レベルの小さな成果のみ",
  "学び曖昧": "「多くのことを学んだ」など抽象的",
  "STAR不完全": "ResultやActionが不明確"
};
```

### Phase 3: 業界特化版 (2-3時間)

#### 3.1 業界別ガクチカ評価軸
- **IT業界**: 技術力、課題解決、チーム開発
- **金融業界**: 数値分析、リスク管理、信頼性
- **商社**: 交渉力、グローバル視点、調整力
- **メーカー**: 物づくり、継続改善、安全意識

## 🛠️ 共通実装ガイド

### 1. スクリプトファイル構成
```
project-root/
├── add_[target]_questions.js      # Q001-Q100追加
├── extract_[target]_answers.js    # 模範解答抽出
├── [target]_bad_database.js       # 悪い例データ
├── add_[target]_bad_answers.js    # 悪い例追加
├── validate_[target]_database.js  # バリデーション
└── [target]_database.json         # 最終成果物
```

### 2. JSON構造統一
```json
{
  "metadata": {
    "title": "[ES/ガクチカ]100問データベース", 
    "type": "es|gakuchika",
    "target_industries": ["IT", "金融", "商社"],
    "evaluation_method": "STAR|PREP"
  },
  "questions": [
    {
      "id": "Q001",
      "question": "質問文",
      "category": "カテゴリ",
      "word_limit": 400,
      "target_industry": ["IT", "金融"],
      "evaluation_points": "評価ポイント",
      "good_answer": "模範解答",
      "bad_answer": "問題のある解答例"
    }
  ]
}
```

### 3. バリデーション項目
- **基本チェック**: ID連続性、必須フィールド
- **文字数チェック**: 制限内におさまっているか
- **内容チェック**: 評価軸への適合性
- **業界チェック**: 対象業界への適合性

## 📚 成功のための重要ポイント

### 1. 悪い例作成の黄金ルール
**❌ 極端な悪い例（非現実的）**
- 「特にやったことないです」
- 「別に何も学ばなかったです」

**✅ 評価ポイント違反型（現実的）**
- 抽象的で具体性なし
- 企業研究が浅い
- 自己中心的な視点のみ
