# 新規テーマ専用リサーチシステム

## 🎯 システム概要
既存の3つのデータベース（習慣・企業・ツール）でカバーできないテーマに対応する新規リサーチシステム。God of Research + 5専門家分析 + Constitutional AI品質保証を活用し、新しい価値ある情報をフィードポスト化。

**対象テーマ**: 既存データベース外の全テーマ
**品質基準**: 既存システムと同等の高品質保証
**最終目標**: 新規データの既存データベースへの統合も視野

### Step 1: テーマ分析・最適キャラクター選択
```
## 1-1: テーマ特性分析

【分析フレームワーク】
```
テーマ: [新規テーマを入力]

【特性分析】
1. 内容性質:
   - 感情・体験重視 → Type001(MISAKI)適合性
   - スキル・実践重視 → Type002(KING)適合性  
   - データ・分析重視 → Type003(KIKUYO)適合性
   - 効率・システム重視 → Type004(TEN)適合性
   - キャリア・成長重視 → Type005(IIDA)適合性

2. ターゲット層:
   - 年齢層: 20代前半/20代後半/30代前半
   - 職業: 学生/新社会人/転職検討者/専門職
   - 関心: 自己啓発/キャリア/効率化/人間関係

3. コンテンツ傾向:
   - 情報提供型/ハウツー型/分析型/体験談型
   - 長期的価値/即効性/エンタメ性
   - データ重視/感情重視/実践重視
```

## 1-2: 最適キャラクター自動選択

【選択アルゴリズム】
```
IF テーマ.emotion_focus > 70% THEN Type001(MISAKI)
ELSE IF テーマ.skill_focus > 70% THEN Type002(KING)  
ELSE IF テーマ.data_focus > 70% THEN Type003(KIKUYO)
ELSE IF テーマ.efficiency_focus > 70% THEN Type004(TEN)
ELSE IF テーマ.career_focus > 70% THEN Type005(IIDA)
ELSE 総合判定による最適キャラクター選択
```

【キャラクター特性確認】
選択されたキャラクターの特性・口調・強み・制約事項を再確認し、
新規テーマとの適合性を最終検証。

## 1-3: リサーチ方向性・範囲設定

【リサーチスコープ定義】
- 調査深度: 表面的/標準的/深掘り
- 情報源範囲: 公開情報/専門情報/最新トレンド
- エビデンスレベル要求: A（最高）/B（高）/C（中）
- 実践可能性重視度: 低/中/高
- 独自性・差別化要求度: 低/中/高
```

### Step 3: Constitutional AI品質保証・エビデンス構築
```
## 3-1: 4原則厳格適用

【1. 科学的根拠性 (Evidence-Based)】
✓ 情報源の権威性・信頼性確認
✓ 統計データ・研究結果の出典明記
✓ 複数ソースでの事実確認・クロスチェック
✓ エビデンスレベル（A/B/C）の明確化

【2. 実践可能性 (Actionability)】
✓ 具体的な実行手順の提示
✓ 必要リソース・コスト・時間の算出
✓ スキルレベル別の対応方法
✓ 期待効果・成功確率の現実的予測

【3. 倫理的配慮 (Ethical Consideration)】
✓ 法的コンプライアンス確認
✓ プライバシー・個人情報保護
✓ 多様性・包摂性への配慮
✓ 環境・社会への影響評価

【4. 透明性 (Transparency)】
✓ 情報源・参考文献完全明記
✓ 調査方法・分析プロセス説明
✓ 限界・制約事項の明示
✓ 利益相反・バイアス可能性開示

## 3-2: Self-Consistency検証システム

【3パス独立検証】
```
パス1: 論理的整合性
- 主張と根拠の整合性確認
- 内部矛盾・論理の飛躍検出
- 因果関係の妥当性検証

パス2: 実用性・現実性  
- 実装可能性の現実的評価
- コスト・効果の妥当性確認
- ターゲット層での適用可能性

パス3: 社会的・倫理的適合性
- 社会的価値・意義の評価
- 多様な立場・視点への配慮
- 長期的影響・持続可能性
```

【品質スコア算出】
各パスの評価を統合し、総合品質スコア（100点満点）を算出。
80点以上で次ステップ進行、79点以下で再分析・改善実施。

## 3-3: 独自価値・差別化要素構築

【独自性確保】
- 既存コンテンツとの差別化ポイント明確化
- 新たな視点・切り口の発見・提示
- オリジナルインサイト・発見の創出
- 実用的価値・読者メリットの最大化

【価値証明】
- なぜこの情報が重要なのかを明確化
- 読者の課題・ニーズへの具体的貢献
- 行動変容・成果創出への道筋提示
- フィードポスト保存・活用価値の創出
```
