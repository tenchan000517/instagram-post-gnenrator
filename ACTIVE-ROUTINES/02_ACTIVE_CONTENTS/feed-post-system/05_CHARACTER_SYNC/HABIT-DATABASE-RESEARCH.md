## 🎯 システム概要
既存の習慣行動データベースシステム（4ステップ起動術式）をフィードポスト生成用に完全統合。habit-behavior-database.json + 9分類DBを活用した高品質コンテンツ生成を実現。

**対応データベース**:
- メインDB: `../../data-systems/03_DATABASE/habit-databases/habit-behavior-database.json` (1,732行)
- 分類DB: `../../data-systems/03_DATABASE/habit-databases/classified-dbs/` (9種)

**対応キャラクター**: Type001(MISAKI), Type002(KING), Type004(TEN), Type005(IIDA)


### Step 4: フィードポスト品質評価・最終調整（統合品質保証）
```
## 4-1: Constitutional AI品質チェック

【必須確認項目】
✓ 科学的根拠性: エビデンスレベルA-B、出典明記
✓ 実践可能性: 具体的手順、コスト・時間明記
✓ 倫理的配慮: プライバシー、多様性配慮
✓ 透明性: データソース明記、限界事項開示

## 4-2: キャラクター整合性チェック

【Type001(MISAKI)】
- 感情共感表現の自然さ
- 体験談・ストーリーの リアリティ
- 安心感・希望感の演出

【Type002(KING)】
- 実践指導の具体性・明確性
- ステップの論理性・実行可能性
- 成果へのコミット感

【Type004(TEN)】
- データ・数値の正確性
- 効率化提案の合理性
- システム思考の一貫性

【Type005(IIDA)】
- キャリア観点の適切性
- 就活・転職リアリティ
- 長期戦略の有効性

## 4-3: フィードポスト適合性評価

【形式チェック】
- 文字数制限: 2200文字以内
- 構成: 導入→本論→結論の流れ
- 視覚要素: 箇条書き、数値、ランキングの活用
- ハッシュタグ: テーマ関連12個（4+4+4構成）

【エンゲージメント要素】
- フック力: 最初の1-2行の引きつけ力
- 共感性: ターゲット層の課題・感情への対応
- 実用性: 即座に行動につながる具体性
- 拡散性: シェア・保存したくなる価値

## 4-4: 品質スコア算出

【評価基準（100点満点）】
- データ正確性: 25点
- キャラクター整合性: 25点
- 実用性・行動喚起: 25点
- エンゲージメント期待値: 25点

【合格基準】80点以上で実投稿可能
【修正指示】79点以下の場合、具体的改善提案
```
