# 次世代Claude_Code引き継ぎ書：IIDA面接質問リール投稿システム

## プロジェクト概要

IIDAのリール投稿とフィード投稿のマスタープロンプトを、面接質問データベースから作成するプロジェクトです。

### 基本情報
- **データベース**: `C:\instagram-course\instagram-post-generator\app\data\interview-question\面接質問データベース.json`
- **総質問数**: 100問（Q001-Q100）
- **カテゴリ数**: 10カテゴリ
- **保存場所**: `INTEGRATED-SYSTEM\LAUNCH\reel-posts`

## 投稿パターン設計

### パターン1: 質問→悪い例→良い例（3問セット）
```
質問1 → 悪い例 → 良い例
質問2 → 悪い例 → 良い例  
質問3 → 悪い例 → 良い例
```

### パターン2: カテゴリ別質問まとめ形式
```
導入：「答えられないと落ちる○○の質問○選」
問題提起：準備不足による面接失敗の回避
カテゴリ別に質問を整理して紹介
最後に解答準備の促し
```

## データベース構造詳細

### スコアリングシステム
- **importance**: 重要度（1-5）合否への影響度
- **frequency**: 出題頻度（1-5）よく聞かれる度
- **difficulty**: 回答難易度（1-5）答えるのが難しい度
- **total_score**: 総合点（importance + frequency + difficulty）

## 重要な設計思想：戦略的重複管理

### 問題となる重複パターン
❌ **悪い重複例**:
- 投稿A「絶対聞かれる3問【第1弾】」: Q001, Q002, Q003
- 投稿B「絶対聞かれる3問【第2弾】」: Q001, Q002, Q004 ← 2/3重複で差別化不足

❌ **グローバル過多使用**:
- Q001が全投稿の70%に出現 ← 避けるべき

### 正しい重複管理
✅ **良い重複例**:
- 投稿A「絶対聞かれる3問【第1弾】」: Q001, Q002, Q003
- 投稿B「絶対聞かれる3問【第2弾】」: Q004, Q005, Q006 ← 完全に異なる質問

✅ **戦略的再使用**:
- 「絶対聞かれる質問」でQ001使用
- 「自己紹介系質問」でQ001再使用 ← 異なる文脈なのでOK

## 設計原則（重要）

### 1. 数字先行設計の禁止
- ❌ 「86投稿作らなければ」という制約
- ✅ 「意味のある投稿を作った結果、○投稿になった」

### 2. 同一タイトルシリーズ内の重複排除
- 同じタイトルで複数投稿作る場合、質問重複を避ける
- 例：「第1弾」「第2弾」で異なる質問セットを使用

### 3. 質問の戦略的価値分析
スコアの戦略的意味：
- **トータル低スコア** = 基礎固め質問
- **重要度高×頻出低** = 差別化質問（「準備してない人は答えられない」）
- **頻出高スコア** = 鉄板必須質問（「絶対聞かれる」）
- **重要度高スコア** = 合否直結質問（「これ答えられないと落ちる」）
